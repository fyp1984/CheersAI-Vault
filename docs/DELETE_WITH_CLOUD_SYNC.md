# 删除文件时同步删除云端文件功能

## 功能概述

在文件管理页面删除本地文件时，可以选择同时删除 FileBay 云端的对应文件。

## 使用方法

### 删除单个文件

1. 在文件管理页面，点击文件行的"删除"按钮
2. 在确认对话框中，会看到一个复选框：**"同时删除 FileBay 云端文件"**
3. 如果勾选此选项，将同时删除：
   - 本地文件
   - FileBay 仓库中的 `masked/文件名` 文件
4. 点击"确认删除"执行操作

### 批量删除文件

1. 勾选要删除的多个文件
2. 点击"批量删除"按钮
3. 在确认对话框中，勾选"同时删除 FileBay 云端文件"
4. 点击"确认删除"
5. 系统会逐个删除本地和云端文件

## 功能特性

### ✅ 智能显示
- 仅在 FileBay 已配置并启用时显示云端删除选项
- 如果 FileBay 未配置，不会显示该选项

### ✅ 灵活选择
- 默认不勾选，避免误删云端文件
- 用户可以自主选择是否删除云端文件
- 支持单个文件和批量文件删除

### ✅ 详细反馈
- 成功删除：显示"本地和云端文件已删除"
- 仅本地删除：显示"本地文件已删除"
- 部分失败：显示"本地文件已删除，但云端删除失败"
- 批量删除：显示成功和失败的数量

### ✅ 错误处理
- 本地删除失败：整个操作失败
- 云端删除失败：本地文件已删除，显示警告信息
- 批量删除时：统计成功和失败的数量

## 技术实现

### 前端实现

```typescript
const handleDelete = (filePath: string) => {
  const fileName = filePath.split(/[\\/]/).pop() || filePath;
  setConfirmDialog({
    open: true,
    title: '确认删除',
    description: `确定要删除文件「${fileName}」吗？此操作不可撤销。`,
    showCloudOption: giteaEnabled, // 仅在 FileBay 启用时显示
    deleteCloud: false, // 默认不勾选
    onConfirm: async () => {
      const shouldDeleteCloud = confirmDialog.deleteCloud;
      
      // 删除本地文件
      await invoke<string>('delete_sandbox_file', { filePath });
      
      // 如果选择了同时删除云端文件
      if (shouldDeleteCloud && giteaEnabled) {
        const remotePath = `masked/${fileName}`;
        await deleteFromGitea(remotePath, `删除文件: ${fileName}`);
      }
    },
  });
};
```

### 云端文件路径映射

本地文件 → 云端文件路径：
- `C:\output\test.pdf` → `masked/test.pdf`
- `C:\output\document.docx` → `masked/document.docx`

## 使用场景

### 场景 1: 清理测试文件
- 测试完成后，删除本地和云端的测试文件
- 勾选"同时删除 FileBay 云端文件"
- 一次性清理干净

### 场景 2: 保留云端备份
- 本地空间不足，需要删除本地文件
- 不勾选云端删除选项
- 保留云端备份，以后可以重新下载

### 场景 3: 批量清理过期文件
- 选择多个过期文件
- 勾选云端删除选项
- 批量清理本地和云端

## 注意事项

### ⚠️ 重要提示

1. **不可撤销**
   - 本地文件删除后无法恢复
   - 云端文件删除后仍保留在 Git 历史中，可通过 Git 恢复

2. **默认行为**
   - 默认不勾选云端删除
   - 避免误删云端文件

3. **权限要求**
   - 需要 FileBay 已配置并启用
   - Token 必须有仓库写入权限

4. **错误处理**
   - 本地删除失败：整个操作失败
   - 云端删除失败：本地已删除，显示警告

5. **批量删除**
   - 逐个删除云端文件
   - 统计成功和失败数量
   - 即使部分失败，也会继续处理其他文件

## 常见问题

### Q: 为什么默认不勾选云端删除？
A: 为了安全起见，避免用户误删云端文件。用户需要明确选择才会删除云端文件。

### Q: 如果云端删除失败怎么办？
A: 本地文件已删除，会显示警告信息。可以稍后在 FileBay 配置页面手动删除云端文件。

### Q: 批量删除时，如果部分云端文件删除失败？
A: 系统会继续处理其他文件，最后显示成功和失败的数量。

### Q: 删除的云端文件可以恢复吗？
A: 可以。文件内容保留在 Git 历史中，可以通过 Git 命令恢复。

### Q: 如果 FileBay 未配置，会显示云端删除选项吗？
A: 不会。只有在 FileBay 已配置并启用时才会显示该选项。
