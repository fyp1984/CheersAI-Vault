# CheersAI Vault - 功能更新 (2026-05-13)

## 🎉 新功能：FileBay 文件删除

### 功能概述

现在可以直接从 CheersAI Vault 应用中删除 FileBay（Gitea）仓库中的文件，无需手动访问 FileBay 网站。

### 主要特性

#### ✨ 核心功能
- **删除单个文件** - 通过文件路径删除仓库中的文件
- **自动 SHA 获取** - 系统自动获取文件版本信息，无需手动操作
- **自定义提交信息** - 可以为每次删除操作添加有意义的 Git commit 消息
- **双环境支持** - 同时支持 UAT 测试环境和标准 Gitea 环境

#### 🎨 用户体验
- **直观的界面** - 简洁易用的删除界面
- **确认对话框** - 删除前需要确认，防止误操作
- **实时反馈** - 操作过程中显示加载状态
- **友好的错误提示** - 清晰的错误消息和解决建议

#### 🔒 安全性
- **权限验证** - 自动检查 FileBay 配置和权限
- **版本控制** - 使用 SHA 验证防止并发冲突
- **Token 认证** - 安全的 Access Token 身份验证

### 使用方法

#### 方法 1: 通过 UI 界面

1. 打开 **FileBay 配置** 页面
2. 确保 FileBay 已配置并启用
3. 滚动到页面底部的 **FileBay 文件管理** 区域
4. 输入要删除的文件路径（例如：`masked/document.pdf`）
5. 可选：输入自定义提交信息
6. 点击 **删除文件** 按钮
7. 在确认对话框中确认删除

#### 方法 2: 在代码中使用

```typescript
import { deleteFromGitea } from '@/services/gitea';

// 删除文件
try {
  const result = await deleteFromGitea(
    'masked/document.pdf',
    '删除已过期的脱敏文件'
  );
  console.log(result); // "文件删除成功"
} catch (error) {
  console.error('删除失败:', error);
}
```

### 使用场景

1. **清理过期文件** - 删除不再需要的脱敏文件
2. **管理存储空间** - 定期清理临时文件和缓存
3. **版本管理** - 删除旧版本文件，保留最新版本
4. **错误修正** - 删除错误上传的文件

- 实现了完整的 Gitea DELETE API 调用
- 自动获取文件 SHA 进行版本验证
- 支持 UAT 环境的特殊处理（通过浏览器 fetch）
- 完善的错误处理和用户友好的错误消息

#### 前端 (TypeScript/React)
- 类型安全的服务层接口
- 用户友好的 UI 组件

- 集成到现有的 FileBay 设置页面

### 注意事项

⚠️ **重要提示**:
- 删除操作会创建新的 Git commit
- 文件内容仍保留在 Git 历史中，可以恢复
- 需要有效的 Access Token 和仓库写入权限
- 删除是即时生效的，请谨慎操作

### 常见问题

**Q: 删除的文件可以恢复吗？**
A: 可以。文件内容保留在 Git 历史中，可以通过 Git 命令恢复。

**Q: 删除操作需要什么权限？**
A: 需要有效的 Access Token，且 Token 必须有仓库的写入权限。

**Q: 如何删除子目录中的文件？**
A: 使用相对路径，例如：`reports/2024/annual-report.pdf`

**Q: 删除失败怎么办？**
A: 查看错误消息，常见原因包括：文件不存在、权限不足、网络问题等。详见[故障排查](./QUICK_START_DELETE_FILE.md#-故障排查)。
