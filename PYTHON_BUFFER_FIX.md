# Python 输出缓冲问题修复

## 🔍 问题诊断

### 症状
- 点击"一键安装" Ollama
- 只显示 "=== Starting Ollama download ==="
- 没有后续的详细日志
- 看起来像卡住了

### 根本原因
**Python 输出缓冲**

Python 默认会缓冲 stdout 输出，这意味着：
- 日志不会立即显示
- 需要积累到一定量或程序结束才会输出
- 在长时间运行的脚本中，用户看不到实时进度

### 验证
手动运行 Python 脚本时可以看到详细日志：
```bash

[2026-04-29 16:01:28] [INFO] ============================================================
[2026-04-29 16:01:28] [INFO] ⚠ 重要提示：
[2026-04-29 16:01:28] [INFO]   - 首次安装需要下载约 1.6GB 文件
[2026-04-29 16:01:28] [INFO]   - 支持断点续传，可随时中断后继续
...
```

但通过 Rust 调用时看不到这些日志。

## ✅ 解决方案

### 修改内容
**文件**: `src-tauri/src/commands/installer.rs`

**添加 `-u` 参数**:
```rust
// 修改前
cmd.arg(&script_path);
for arg in args {
    cmd.arg(arg);
}

// 修改后
// 添加 -u 参数禁用输出缓冲
cmd.arg("-u");
cmd.arg(&script_path);
for arg in args {
    cmd.arg(arg);
}
```

### `-u` 参数的作用
```bash
python -u script.py
```

- **禁用 stdout 和 stderr 缓冲**
- **实时输出日志**
- **适用于长时间运行的脚本**
- **不影响脚本功能**

## 🎯 技术细节

### Python 缓冲机制
Python 有三种缓冲模式：
1. **全缓冲** (默认) - 缓冲区满或程序结束时才输出
2. **行缓冲** - 遇到换行符时输出
3. **无缓冲** (`-u`) - 立即输出

### 为什么需要 `-u`？
在我们的场景中：
- 脚本运行时间长（10-30分钟）
- 需要实时显示进度
- 用户需要知道脚本在运行而不是卡住
- 缓冲会导致日志延迟显示

### 其他解决方案（未采用）
1. **在 Python 中设置**:
   ```python
   import sys
   sys.stdout = os.fdopen(sys.stdout.fileno(), 'w', 0)  # 无缓冲
   ```
   缺点：需要修改脚本

2. **使用环境变量**:
   ```rust
   cmd.env("PYTHONUNBUFFERED", "1");
   ```
   缺点：可能不如 `-u` 可靠

3. **手动刷新**:
   ```python
   print("...", flush=True)
   ```
   缺点：需要在每个 print 语句中添加

**选择 `-u` 的原因**:
- ✅ 简单可靠
- ✅ 不需要修改脚本
- ✅ Python 官方推荐
- ✅ 一行代码解决
