# CheersAI Desktop macOS 测试安装指引

本文档适用于当前 `portable.dmg` 测试包，例如：

- `CheersAI Desktop_0.1.7_aarch64_portable.dmg`

该安装包面向 `Apple Silicon` Mac（M1/M2/M3/M4 系列）测试设备。

## 1. 安装前说明

- 当前测试包采用 `portable` 过渡打包方案。
- 该方案已经修复“已损坏，无法打开，移到废纸篓”的问题。
- 由于当前版本仍未使用 Apple Developer ID 正式签名，也未完成 Apple notarization，首次打开时，macOS 仍可能提示“无法验证开发者”或阻止直接打开。
- 这是测试包阶段的正常现象，不代表安装包损坏。

## 2. 安装步骤

1. 双击打开 `CheersAI Desktop_*.dmg`。
2. 在弹出的安装窗口中，将 `CheersAI Desktop.app` 拖动到 `Applications`。
3. 等待复制完成后，打开 `Applications` 文件夹。
4. 找到 `CheersAI Desktop.app`，先不要连续双击多次。

## 3. 首次打开方式

推荐使用以下任一方式完成首次打开：

### 方式 A：右键打开

1. 在 `Applications` 中找到 `CheersAI Desktop.app`。
2. 右键应用图标，选择 `打开`。
3. 如果系统再次提示安全确认，点击 `打开`。

说明：

- 右键打开通常比直接双击更容易触发一次性放行。
- 成功一次后，后续通常可直接双击启动。

### 方式 B：系统安全设置放行

如果系统提示无法打开、无法验证开发者，按以下步骤处理：

1. 打开 `System Settings`。
2. 进入 `Privacy & Security`。
3. 向下滚动到安全提示区域。
4. 找到与 `CheersAI Desktop` 相关的阻止提示。
5. 点击 `Open Anyway`。
6. 回到应用，再次打开。

说明：

- 放行后，系统会将该应用记录为一次例外。
- 之后通常可以正常双击打开。

## 4. 首次启动后的预期现象

首次启动后，可能出现以下情况，均属正常：

- 应用启动稍慢：桌面端会初始化本地数据库与运行环境。
- 云端工作区需要联网：`CheersAI` 在线工作区依赖网络连接。
- OCR 或 AI 服务未安装：如页面提示需安装运行时，请按应用内引导操作。

## 5. 常见问题

### 1. 仍提示“已损坏，无法打开”

请先确认：

- 使用的是带 `_portable.dmg` 后缀的测试包。
- 不是旧的原始 `dmg` 包。

如果仍出现该提示，请把以下信息反馈给研发：

- macOS 版本
- Mac 芯片类型（Intel / Apple Silicon）
- 安装包完整文件名
- 是否通过 AirDrop、微信、浏览器、网盘等方式传输

建议补充一张报错截图。

### 2. 提示“无法验证开发者”

这是当前测试包阶段的正常现象。

处理方式：

- 优先使用“右键 -> 打开”
- 或在 `Privacy & Security` 中点击 `Open Anyway`

### 3. 双击没有反应

请按以下顺序排查：

1. 等待 5-10 秒，避免重复启动多个实例。
2. 到 `Applications` 中右键 `打开`。
3. 到 `Privacy & Security` 中查看是否有新的拦截提示。

### 4. 提示与系统架构不兼容

当前测试包主要面向 `Apple Silicon`。

如果测试设备是 Intel Mac，请先联系研发确认是否提供 `x86_64` 版本。

## 6. 向研发反馈时请附带的信息

如安装失败，请尽量一次性提供以下信息：

- 设备型号
- macOS 版本
- 芯片类型
- 安装包完整名称
- 报错截图
- 是在双击、右键打开，还是点击 `Open Anyway` 时失败

## 7. 当前测试包结论

- `portable.dmg`：当前推荐测试包
- 原始 `dmg`：不要继续使用
- 首次出现安全提示：属测试阶段正常现象
- 能通过 `Open Anyway` 或右键打开：说明安装包链路基本正常
