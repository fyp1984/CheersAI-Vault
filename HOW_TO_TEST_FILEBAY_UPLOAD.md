# 如何测试 FileBay 上传功能

## 前提条件
✅ 开发服务器已启动（正在运行）
✅ 应用已打开

## 测试步骤

### 步骤 1: 配置 FileBay

1. 在应用中点击左侧菜单的 **"FileBay 设置"**
2. 点击 **"导入配置"** 按钮
3. 选择 `filebay-config.json` 文件
4. 配置应该自动加载：
   - URL: `https://uat-filebay.cheersai.cloud`
   - 用户: `admin_cheersai_cloud_de8df0`
   - 仓库: `workspace`

### 步骤 2: 测试连接

1. 点击 **"测试连接"** 按钮
2. 观察结果：
   - ✅ 成功：显示"连接成功！FileBay 服务器可访问"
   - ❌ 失败：显示具体错误信息

### 步骤 3: 检查/创建仓库

1. 查看仓库状态
2. 如果仓库不存在，点击 **"创建仓库"** 按钮
3. 等待创建完成

### 步骤 4: 启用上传功能

1. 找到 **"启用 FileBay 上传"** 开关
2. 确保开关是打开状态（绿色）

### 步骤 5: 准备测试文件

1. 点击左侧菜单的 **"文件处理"**
2. 上传一个测试文件（任意文本文件或文档）
3. 选择脱敏规则
4. 点击 **"开始脱敏"**
5. 等待脱敏完成

### 步骤 6: 测试上传

1. 脱敏完成后，点击左侧菜单的 **"文件管理"**
2. 找到刚才脱敏的文件
3. 点击文件右侧的 **"上传"** 按钮（绿色上传图标）
4. 观察提示消息：
   - ✅ 成功：显示"[文件名] 上传成功"（绿色提示）
   - ❌ 失败：显示"上传失败: [错误信息]"（红色提示）

### 步骤 7: 查看日志

打开开发服务器的终端窗口，查看日志输出：

**成功的日志示例**:
```
✓ 文件已存在，SHA: abc123...
执行更新操作...
```

**失败的日志示例**:
```
上传失败(browser): [具体错误]
```

## 常见问题排查

### 问题 1: "请先在 FileBay 设置中完成配置"

**原因**: FileBay 功能未启用

**解决方案**:
1. 进入 FileBay 设置
2. 确保配置已导入
3. 打开"启用 FileBay 上传"开关

### 问题 2: "FileBay 功能未启用"

**原因**: 后端配置中 enabled 为 false

**解决方案**:
1. 进入 FileBay 设置
2. 重新导入配置文件
3. 或手动启用上传功能

### 问题 3: "认证失败: Token 无效"

**原因**: Token 过期或错误

**解决方案**:
1. 访问 FileBay 网站重新生成 Token
2. 下载新的配置文件
3. 重新导入配置

### 问题 4: "仓库不存在"

**原因**: 仓库未创建

**解决方案**:
1. 在 FileBay 设置中点击"创建仓库"
2. 等待创建完成
3. 重新测试上传

### 问题 5: "上传失败: [网络错误]"

**原因**: 网络连接问题

**解决方案**:
1. 检查网络连接
2. 确认可以访问 `uat-filebay.cheersai.cloud`
3. 检查防火墙设置

## 验证上传成功

### 方法 1: 查看提示消息
- 绿色提示："[文件名] 上传成功"

### 方法 2: 查看后端日志
- 终端中应该看到成功的日志

### 方法 3: 访问 FileBay 网站
1. 打开浏览器访问: `https://uat-filebay.cheersai.cloud`
2. 登录账号
3. 进入 `workspace` 仓库
4. 查看 `masked/` 目录
5. 确认文件已上传

## 收集错误信息

如果上传失败，请收集以下信息：

### 1. 错误提示消息
- 应用中显示的红色错误提示

### 2. 后端日志
- 开发服务器终端中的错误日志
- 查找包含 "上传失败" 的行

### 3. 浏览器控制台
- 按 F12 打开开发者工具
- 切换到 Console 标签
- 查找红色错误消息

### 4. 网络请求
- 在开发者工具的 Network 标签
- 查找失败的请求
- 查看请求详情和响应

### 5. 配置信息
```bash
# 查看配置文件
cat filebay-config.json
```
